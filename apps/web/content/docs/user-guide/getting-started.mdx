## The operator path in one page

Keppo works best when you treat setup as four explicit stages:

1. Create or select a workspace.
2. Connect the providers your automation needs.
3. Generate a draft automation and review the proposed actions.
4. Run it with approval gates still intact, then decide what should stay manual and what can be auto-approved later.

## First session checklist

- Sign in and let Keppo create your default workspace if one does not exist yet.
- Open **Integrations** and connect the smallest set of providers needed for your first use case.
- Start the guided builder from the dashboard or command palette.
- Review generated prompts, questions, and approval behavior before activating anything.

<Callout title="Start small" tone="warning">
Do not begin by wiring every provider in your stack. Keppo is most understandable when the first automation has one clear goal, one or two integrations, and human approvals still turned on.
</Callout>

## What a healthy first automation looks like

A good first automation usually has these traits:

- It saves repeated operator time.
- A mistaken write is reversible.
- The approval reviewer can tell, in plain language, what will happen next.
- The success condition is obvious from the run detail page.

Examples:

- Drafting Gmail replies for a shared inbox while requiring approval before send.
- Commenting on GitHub issues with a prepared response and escalation note.
- Posting structured Slack updates after a billing or incident workflow completes.
