## What notifications should do

Notifications exist to move work forward:

- tell reviewers that approvals are waiting
- surface failures that need operator attention
- confirm user-visible state changes

They should not duplicate dashboard state without adding urgency or actionability.

## Recommended channels

- In-app notifications for general workflow awareness
- Email for actions that cannot wait until the next dashboard visit
- Push only when the team is actually using it as an operational channel

## Operational expectations

- Unread badges should represent real unread work.
- Duplicate delivery attempts should be deduplicated, not multiplied.
- Failed channels should degrade clearly instead of silently disappearing.
