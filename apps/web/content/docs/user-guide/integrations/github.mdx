## Best fit

GitHub is the right connector when your automation needs to:

- inspect issues or pull requests
- draft comments or labels
- trigger or inspect workflows
- coordinate approvals around engineering changes

## Good GitHub use cases

- Route bug reports with a prepared response and label set.
- Draft release-note or triage comments that still need approval before posting.
- Trigger follow-up automation after CI or repository events.

## Guardrails that matter

- Keep repository allowlists explicit.
- Treat merge or destructive write tools as high-risk until proven otherwise.
- Use workspace boundaries to separate internal engineering workflows from customer-facing support repos.
