## The automation lifecycle

Keppo automation work usually flows through these stages:

1. Draft the automation from a prompt or builder flow.
2. Answer setup questions and connect any missing provider requirements.
3. Review the resulting draft, especially write actions and approval behavior.
4. Activate it.
5. Inspect runs, approvals, and failures over time.

## Why Keppo starts from drafts

Drafts exist so operators can see the proposed behavior before an automation touches real systems. That is especially important when the flow reaches Gmail, Stripe, GitHub, or a private API.
