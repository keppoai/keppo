## Runtime layout

The important deployment seams are:

- `apps/web` for the live HTTP surface and typed server functions
- `convex/` for the active Convex runtime
- `cloud/` for shared billing, scheduler, and sandbox modules

`apps/web` builds with Vite plus Nitro using the `vercel` preset. Production output lands in `apps/web/.vercel/output/`.

## Boot expectations

API startup validates env with a schema. Missing required secrets should fail at boot instead of degrading into half-configured behavior.

The most important variables are:

- `KEPPO_URL`
- `CONVEX_URL`
- `KEPPO_CONVEX_ADMIN_KEY`
- `BETTER_AUTH_SECRET`
- `KEPPO_MASTER_KEY`
- `OPENAI_API_KEY` when bundled gateway mode is not configured
- `KEPPO_LLM_GATEWAY_URL`, `KEPPO_LLM_GATEWAY_MASTER_KEY`, and `KEPPO_LLM_GATEWAY_TEAM_ID` when bundled gateway mode is configured
- `KEPPO_CRON_SECRET`

## Deployment-path validation

When you need an end-to-end deployment-path check instead of normal local dev, use Docker Compose:

```bash
docker compose up --build
docker compose down
```

## Practical deployment advice

- Keep dashboard origin and auth callback hosts aligned.
- Keep client and server Convex URLs on the same hostname family in local dev.
- Treat internal cron, queue, and webhook routes as fail-closed surfaces.
