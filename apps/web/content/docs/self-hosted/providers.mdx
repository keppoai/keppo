## Google

- Configure OAuth client credentials.
- Keep callback hosts aligned with the active deployment origin.
- For Gmail trigger flows, verify watch or polling prerequisites before treating inbox automations as healthy.

## GitHub

- Configure OAuth client credentials and webhook secrets.
- Keep repository access explicit.
- Separate login credentials from integration credentials; signing in with GitHub is not the same as connecting the GitHub provider for automation.

## Stripe

- Configure OAuth client credentials and billing webhook secrets.
- Enable the billing webhook events your deployment expects, including subscription lifecycle updates.
- Treat billing writes as high-signal approval surfaces until the team is comfortable automating them.

## LinkedIn

- Configure LinkedIn OAuth client credentials and keep the integration callback on the active deployment origin.
- Expect product-family access to vary by app approval; a successful connect does not guarantee Community Management, Ads, Sales Navigator, or Talent APIs are all available.
- Start with the low-level request/response tools and request extra scopes only for the LinkedIn products your app is actually approved to use.

## Custom and other providers

- Keep catalog-backed providers visible even when env is incomplete.
- For custom or private integrations, document the action surface and network expectations before rollout.
