## Default mindset

Keppo should fail closed on sensitive boundaries. That applies to:

- auth and session handling
- internal cron and queue routes
- provider webhooks
- sandbox execution
- public diagnostics

## Concrete expectations

- Missing required secrets should fail boot, not degrade silently.
- Webhook handlers verify signatures before mutating state.
- Public and anonymous routes must stay sanitized.
- Sandbox and networking rules should be explicit, especially for local-only modes.

## When to stop and read the deeper rules

If your change touches auth, webhooks, billing, internal routes, sandbox behavior, or GitHub Actions that run coding agents, go back to the repo rule files before you continue. This section is a contributor summary, not a replacement for those rules.
